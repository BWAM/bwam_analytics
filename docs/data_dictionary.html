4  Data Dictionary

Author

Zachary M. Smith

The data dictionary below describes data from the WQMA Oracle database and the WQMA Analytical Data Store (WQMA parquet files).

Important

The data dictionary empowers data users to quickly become familiar with a data set. Although the table is simple, it is critical to operations that this table remain up-to-date, accurate, and easy to understand.

If you find any errors or do not find the documentation clear, please open a GitHub Issue and we can collectively work towards a solution.

The dictionary provides:

  1. table_name: The name of the table in the WQMA Oracle Database where the column_name is stored.

  2. column_name: The name of the column in the data store of interest. The naming convention is the same between the WQMA Oracle Database and WQMA Analytical Data Store.

  3. examples: Provides examples of the data found in a column. If more than 20 unique values are represented in the column, then 20 example character strings will be provided. If 20 or less unique values are represented in the column, all 20 values are provided.

  4. definition: A description of what the column represents.

  5. db_data_type: The columns data type in the Oracle Database.

  6. character_limit: If present, the numeric value represents the maximum character string length that is allowed in VARCHAR or character column of the Oracle database.

  7. not_null: Indicates if the column can (FALSE) or cannot (TRUE) contain NULL or NA values.

  8. surrogate_key: Indicates if a column represents a unique sequential number ID generated by the Oracle database (TRUE) or a value provided by the program (FALSE).

  9. primary_key: Indicates if a column represents the unique identifier for a table (TRUE) or not (FALSE). Multiple columns can be listed as primary keys, which together form a composite key that uniquely identifies a row in the table.

  10. foreign_key: Indicates if a column has a relationship to another table (TRUE) or not (FALSE). There can be multiple columns in a single table that act as foreign keys to a single table. There can be multiple foreign key relationships, but a single column should only relate to a single parent_table.

  11. parent_table: Provides the name of the table that a foreign key column relates to.
